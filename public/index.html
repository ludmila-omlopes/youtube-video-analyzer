Product surfaces

The same analysis engine, exposed three ways.

The repo already behaves more like a platform than a single MCP package. The right move is to keep MCP, but stop letting it define the whole product story.

01

Local MCP for agent-native workflows

Registry-ready stdio for desktop clients, deep follow-up sessions, and the blocking tools that only make sense close to local compute.

  • Desktop and registry-friendly entrypoint
  • Uploaded-file long-video path
  • Follow-up analysis by session ID
02

Remote MCP for hosted clients

Streamable HTTP for agents that need a public endpoint, bearer auth, and a remote-safe way to kick off long-running analysis jobs.

  • Public endpoint at /api/mcp
  • OAuth-protected resource metadata when enabled
  • Queue-backed long-video polling flow
03

Platform core for future creator products

Metadata normalization, chunk planning, Gemini orchestration, credits, jobs, and account boundaries already point toward a broader YouTube workflow platform.

  • Reusable analysis service
  • Worker-friendly runtime shape
  • Natural path to web app, API, and monetization

Deploy shape

Render is the production path. Vercel is optional.

The current runtime already wants a hosted MCP endpoint, a background worker, Redis, and an admin surface. That architecture maps cleanly to Render and only partially to Vercel.

Primary

Render Recommended

Canonical production setup for this repo because it matches the whole operating shape: web service, worker, Redis-backed BullMQ, and an admin dashboard.

  • Runs npm run start:http for the hosted endpoint
  • Runs npm run start:worker off the request path
  • Uses Render Key Value for BullMQ state and remote jobs
Optional

Vercel Thin adapter

Keep only if you want a stateless MCP HTTP adapter or preview surface. It is not the canonical deployment target for the full platform runtime.

  • api/mcp.ts remains a small compatibility layer
  • vercel.json is optional, not production-critical for Render
  • Do not let the Vercel path drive product positioning
